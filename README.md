# Cyclistic Bike-Share Data Analysis Capstone Project

## 🚴‍♂️ About the Project
This repository contains my **Google Data Analytics Professional Certificate Capstone Project**. The analysis focuses on **Cyclistic**, a fictional bike-share company in Chicago. The core business goal is to design marketing strategies aimed at converting casual riders into annual members.

---

## 📊 Business Task
Analyze historical bike trip data to understand **how annual members and casual riders use Cyclistic bikes differently**, providing actionable, data-driven insights to guide the marketing team.

---

## 🛠️ Tools Used
* **Google Cloud BigQuery (SQL):** For large-scale data cleaning, transformation, and aggregation across millions of rows.
* **R (RStudio):** For statistical data validation, advanced data manipulation, and visualization orchestration.
* **ggplot2 / Power BI:** For designing high-quality, professional standalone charts and integrated dashboards.

---

## 📂 Repository Structure
* `/01_data_cleaning_and_table_creation.sql`: Full SQL script for data preparation, handling missing values, and engineering time-based metrics.
* `/02_final_metrics_extraction.sql`: SQL queries for calculating aggregation metrics (averages, trip counts, distributions).
* `/cyclistic_analysi.R`: R script used to build and arrange the data visualization pipeline.
* `/cyclistic_r_dashboard.png`: High-resolution final visual dashboard showing key trends.
* `/Cyclistic_Bike_Share_Analysis_Q1_2024.pdf`: The comprehensive executive data analysis report.

---

## 💡 Key Insights Captured
1. **Usage Patterns:** Annual members utilize the bike-share system consistently throughout the week, primarily for daily commuting (peak hours correspond to 8 AM and 5 PM). Casual riders peak heavily on weekends for leisure.
2. **Ride Duration:** Casual riders log significantly higher average ride lengths compared to annual members across all days of the week.
